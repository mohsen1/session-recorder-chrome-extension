Session Recorder — Demo & Test Page

Start a recording with the extension, then use the buttons below to drive the capture pipeline. Each control is labelled with the event type it produces. Network calls may 404 or fail offline — that still emits a net-request event, which is the point.

Network — fetch requests net-request

GET has no body; POST sends a JSON body (watch redaction on the password/token keys). Both may fail offline.

Request collapsing — polling net-request → collapsed

Fires 12 identical GET requests to the same endpoint in quick succession. The trimmer (L2+) should collapse these into one representative request plus a ×N similar marker.

Console console

console.log/info/warn become console events; repeated identical logs test coalescing.

Errors error / console

console.error → error event (origin: 'console' via Log.entryAdded). The uncaught exception → error event (origin: 'exception'). A rejected fetch also links an error.

Form & input input / click / key

The password field must be redacted in input events (redacted: true, value replaced with the marker). Typing and Enter produce input/key events.

File upload file-captured

Choosing a file triggers the file-capture content script. Small files are captured whole; oversized files become metadata-only.

Scrolling scroll

Scroll inside this container to emit scroll events. Runs of consecutive scrolls are coalesced by the trimmer.

Activity log page-local

A local echo of what this page did (not part of the recording).
